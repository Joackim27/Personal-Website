Introduction / Project Overview / Objectives

This project, entitled "Joackim Agno Portfolio," is a web development project aimed at showcasing a professional portfolio by demonstrating the ability to develop a website using HTML, CSS, JavaScript, Bootstrap, Node, and React. This website features various sections, such as a personal profile, photo album, technical articles, and professional information and services. Each section is designed to highlight different aspects of my skills, achievements, and capabilities, providing a view of my personal and professional journey as a Computer Engineer. This project integrates advanced web technologies to create a dynamic, responsive, and user-friendly personal website. Moreover, the website implements password-secured pages and server-side implementations. The primary objective of this project is to create a platform that effectively showcases my personal and professional achievements. Its key functionalities include secure access using Node.js and AJAX for handling server-side operations and asynchronous data exchanges, such as file uploads and content loading. Additionally, the technical articles section utilizes data visualization techniques with Google Charts to present data clearly. The professional services section includes an interactive form that processes job orders and provides quotations based on user inputs. Overall, the project ensures consistent design and high usability across all sections using Bootstrap for a unified theme.

Other Requirements
  • Node.JS
  • React.JS
  • First, I installed Node.js, then I opened my terminal with the path in the folder where I wanted to store my Node files. Next, the command "npm init -y" is used to initialize my package.json. This is a quick setup to have a package.json without manually typing it. Next, I installed express-fileupload and nodemon. Express-fileupload is for handling file uploads, and nodemon is for the constant refresh of my server. In my package.json, my main file is "app.js", and I also changed the "scripts" part: the "start" script to "node app.js" and the "dev" script to "nodemon app.js". Next, I created the app.js file with express and cors as required and set the port to 3500. Generally, for the file upload, its HTML file is in the file_upload_node folder. It handles the submission button for the file upload. I also added middleware for my file upload, which includes a file extension limiter, file size limiter, and file existence checker. If the file meets all the requirements of the middleware, I will fetch the Node server and get the file. For the HTML loader using Node.JS, it is in the same folder as the file upload. The only difference is how they are fetched. File uploads are fetched on port/upload, while the HTML is fetched on port/content.
  • For the React.JS, I created the app and generated the package.json with the command "npx create-react-app article5". Then, in the directory article5, I inputted the command "npm start" which initializes the server as running. I added a components folder in my src folder. The components folder have my JS elements to be called by my App.js. I have two components which are ArticleFooter.js and ArticleSection.js.
